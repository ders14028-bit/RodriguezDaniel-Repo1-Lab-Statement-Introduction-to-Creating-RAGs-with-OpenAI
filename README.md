# LangChain Tutorial (Step by Step) using Grok

**Student:** Daniel Rodriguez

This repository contains a practical notebook implementation of a LangChain tutorial using **Llama 3.1 (8B) through Groq**.  
The notebook demonstrates the core LCEL workflow through a Q&A assistant example: direct model calls, parsing, prompt templating, pipeline composition, and streaming output.

---

## Overview

The project demonstrates building a reusable Q&A assistant pipeline in LangChain:

```text
User Question + Topic
	↓
ChatPromptTemplate
	↓
ChatGroq (llama-3.1-8b-instant)
	↓
StrOutputParser
	↓
Final text output / streamed tokens
```

### Main building blocks

| Component | Purpose |
|---|---|
| `ChatGroq` | Connects LangChain to Groq-hosted Llama 3.1 |
| `SystemMessage` / `HumanMessage` | Sends structured role-based chat input |
| `ChatPromptTemplate` | Creates parameterized prompts (`{topic}`, `{question}`) |
| `StrOutputParser` | Converts model responses into plain strings |

---

## What is implemented in `Notebook.ipynb`

1. **Environment preparation**
	- Installs required packages with `%pip`
	- Loads API key from `.env` when available
	- Requests `GROQ_API_KEY` securely if missing

2. **Model initialization**
	- Creates `ChatGroq(model="llama-3.1-8b-instant")`

3. **Direct invocation**
	- Sends `SystemMessage` + `HumanMessage` to create a weather assistant

4. **Output parsing**
	- Uses `StrOutputParser` to normalize output text

5. **Prompt templates**
	- Builds a reusable Q&A prompt with topic and question variables

6. **Full LCEL chain**
	- Composes: `template | model | parser`
	- Runs multiple question-answer examples (weather, geography, etc.)

7. **Streaming**
	- Streams token output in real time with `.stream()`

## Key Concepts

### LCEL (LangChain Expression Language)

LCEL is the pipe-based syntax in LangChain that connects components into executable chains:

```python
chain = prompt | model | parser
result = chain.invoke({"topic": "weather", "question": "What causes rain?"})
```

The `|` operator passes the output of one component as input to the next, similar to Unix pipes. This pattern enables modular, composable workflows.

### Prompt Templates

Templates separate prompt logic from input data, making prompts reusable and testable:

```python
template = ChatPromptTemplate.from_messages([
    ("system", "You are a helpful assistant that answers questions about {topic}."),
    ("user", "{question}")
])
```

Variables like `{topic}` and `{question}` are filled at runtime with `.invoke()`.

### Output Parsers

Parsers transform raw model output into structured formats. The `StrOutputParser` extracts plain text:

```python
parser = StrOutputParser()
chain = model | parser
text_only = chain.invoke(messages)  # Returns str, not AIMessage
```

This simplifies downstream processing when you only need the content, not metadata.

### Streaming

Instead of waiting for the full response, `.stream()` emits tokens as they're generated:

```python
for chunk in chain.stream(input_data):
    print(chunk, end="", flush=True)
```

This improves perceived latency in chat-like interfaces.

### Message Types

LangChain uses role-based messages to structure conversations:

- `SystemMessage`: Sets the model's behavior/context
- `HumanMessage`: Represents user input
- `AIMessage`: The model's response (returned by `.invoke()`)

These align with the chat completion API format used by most LLMs.
